# 图书馆座位管理系统 - 手动清零功能说明

## 功能概述

### 1. 手动清零功能
管理员可以在管理后台一键清空所有当日预约记录，将所有座位状态重置为"空闲"。

### 2. 功能特点
- ✅ **权限控制**：只有管理员账号登录后才能看到"一键清零"按钮
- ✅ **演示友好**：适合课程作业展示，演示前一键清空，随时恢复全新状态
- ✅ **数据保留**：违规记录不受影响，只清空座位和预约相关数据
- ✅ **实时同步**：清零后前端座位地图和统计数字同步刷新

### 后端实现

#### ReservationService.java - 清零API
```java
@Transactional
public Map<String, Object> resetAllReservations(Long adminId) {
    int deletedCount = 0;

    List<Reservation> activeReservations = this.list(
        new QueryWrapper<Reservation>()
            .in("status", "pending", "active", "away")
    );

    for (Reservation reservation : activeReservations) {
        seatService.releaseSeat(reservation.getSeatId());
        this.removeById(reservation.getId());
        deletedCount++;
    }

    return Map.of(
        "success", true,
        "message", "清零成功！已清空" + deletedCount + "条预约记录",
        "deletedCount", deletedCount
    );
}
```

## 定时任务说明（可选）

### 1. 定时清零逻辑
系统中已预留每日凌晨自动清零的定时任务代码，但默认**未启用**。

### 2. 为什么默认不启用？
- **课程作业场景**：通常只需要手动演示功能
- **数据持久性**：保留数据便于查看历史记录
- **服务依赖**：定时任务依赖后端服务持续运行

### 3. 如何启用定时清零？

#### 方案一：Spring Boot定时任务（推荐）
在后端添加定时任务配置：

```java
// ReservationScheduler.java
@Scheduled(cron = "0 0 0 * * ?") // 每天凌晨0点执行
public void cleanupExpiredReservations() {
    reservationService.cleanupExpiredReservations();
    reservationService.resetAllReservations(null);
}
```

#### 方案二：数据库定时任务
在MySQL中创建事件：

```sql
-- 每天凌晨0点执行清零
CREATE EVENT reset_seats_event
ON SCHEDULE EVERY 1 DAY
STARTS CURRENT_TIMESTAMP
DO
BEGIN
    DELETE FROM reservation WHERE status IN ('pending', 'active', 'away');
    UPDATE seat SET status = 'available', user_id = NULL;
END;
```

### 4. 定时任务的限制

⚠️ **重要提醒**：
- 只有后端服务在运行时定时任务才会执行
- 关机、断电、服务器重启都会导致定时任务不执行
- 不上线、不24小时开机不影响系统正常使用
- 手动清零功能可以随时替代定时任务

### 5. 推荐做法

#### 课程作业演示
- ✅ 使用手动清零功能替代定时任务
- ✅ 演示前一键清零，恢复全新状态
- ✅ 更灵活，随时可操作

#### 生产环境
- ✅ 可以启用定时清零任务
- ✅ 需要确保服务器24小时运行
- ✅ 需要配置定时任务监控和告警

## 注意事项

### 1. 操作不可逆
⚠️ **警告**：清零操作会立即删除所有预约记录，且**不可撤销**！
- 请在确认无误后再执行清零
- 建议在非工作时间执行
- 重要数据请提前备份

### 2. 权限控制
- 手动清零功能仅对管理员开放
- 普通用户无法执行清零操作
- 确保管理员账号密码安全

### 3. 数据保留策略
清零时保留以下数据：
- ✅ 违规记录（违规记录独立管理）
- ✅ 用户账号和权限
- ✅ 座位基础信息（座位号、楼层、区域等）

清零时删除以下数据：
- ❌ 所有预约记录
- ❌ 所有使用记录
- ❌ 所有暂离记录

### 4. 性能考虑
- 清零操作使用事务保证数据一致性
- 大量预约记录时可能需要几秒钟
- 建议在低峰期执行清零操作

## 常见问题

### Q1: 清零后还能恢复数据吗？
**答**：不能。清零操作会彻底删除预约记录，且不可撤销。请谨慎操作。

### Q2: 普通用户可以执行清零吗？
**答**：不可以。只有管理员账号登录后才能看到并执行清零操作。

### Q3: 清零会删除用户账号吗？
**答**：不会。清零只删除预约记录，不影响用户账号和权限。

### Q4: 违规记录会被清零吗？
**答**：不会。违规记录独立管理，清零时保留。

### Q5: 如何启用每日定时清零？
**答**：需要在后端添加定时任务配置，并确保服务器24小时运行。具体方法见上文"定时任务说明"部分。

### Q6: 定时任务没执行怎么办？
**答**：定时任务依赖后端服务运行。如果服务关机或重启，定时任务不会执行。可以使用手动清零功能作为替代。
